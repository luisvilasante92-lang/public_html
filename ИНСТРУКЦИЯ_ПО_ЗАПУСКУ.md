# Инструкция по запуску
## Личный кабинет LVR Music Publishing

---

## 1. Требования

- **Python 3.10** или выше  
- Установленный **pip**

Проверка версии Python:
```bash
python --version
```

---

## 2. Установка

### Шаг 1. Открыть папку проекта

Перейдите в каталог проекта в терминале (PowerShell или cmd):

```bash
cd путь\к\папке\LKSYSTEM
```

### Шаг 2. Создать виртуальное окружение

**Windows (cmd):**
```bash
python -m venv venv
venv\Scripts\activate
```

**Windows (PowerShell):**
```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
```

**Linux / macOS:**
```bash
python3 -m venv venv
source venv/bin/activate
```

После активации в начале строки должно появиться `(venv)`.

### Шаг 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### Шаг 4. Настроить окружение

Скопируйте файл с примером переменных окружения:

**Windows:**
```bash
copy .env.example .env
```

**Linux / macOS:**
```bash
cp .env.example .env
```

Файл `.env` можно оставить как есть для локальной разработки. Для продакшена в нём нужно задать свой `SECRET_KEY` и при необходимости `DATABASE_URL`.

---

## 3. Запуск в режиме разработки

Выполните в терминале (с активированным `venv`):

```bash
python run.py
```

В консоли должно появиться что-то вроде:
```
 * Running on http://127.0.0.1:5000
 * Debug mode: on
```

Откройте в браузере: **http://127.0.0.1:5000**

---

## 4. Первый вход (режим разработки)

В режиме разработки используется упрощённый вход без внешнего сервиса авторизации.

### 4.1. Создать тестовые данные

В браузере откройте:

**http://127.0.0.1:5000/dev-setup**

Будут созданы:
- пользователь **admin** с паролем **Admin123!**
- пользователь **artist** с паролем **Artist123!**
- пользователь **label** с паролем **Label123!**
- список платформ распространения по умолчанию

### 4.2. Войти в систему

Перейдите по ссылке:

**http://127.0.0.1:5000/dev-login**

Введите, например:
- **Логин:** `admin`  
- **Пароль:** `Admin123!`

После входа откроется главная страница (дашборд).

---

## 5. Остановка сервера

В терминале, где запущен `python run.py`, нажмите:

**Ctrl + C**

---

## 6. Запуск для production (через Gunicorn)

Для развёртывания на сервере:

```bash
pip install gunicorn
gunicorn -w 4 -b 127.0.0.1:8000 "wsgi:app"
```

- Приложение будет доступно на порту **8000**.  
- Перед этим в `.env` задайте `FLASK_ENV=production`, надёжный `SECRET_KEY` и при необходимости `DATABASE_URL` для PostgreSQL.

---

## 7. Частые проблемы

| Проблема | Решение |
|----------|---------|
| `python` не найден | Используйте `py` (Windows) или `python3` (Linux/macOS), либо установите Python с python.org |
| Ошибка при `pip install` | Обновите pip: `python -m pip install --upgrade pip`, затем снова `pip install -r requirements.txt` |
| Страница не открывается | Убедитесь, что в консоли нет ошибок и сервер запущен на http://127.0.0.1:5000 |
| Нет кнопки «Войти» / редирект на внешний auth | В dev-режиме используйте **/dev-login** и предварительно выполните **/dev-setup** |
| База данных не создаётся | При первом запуске `python run.py` папка `instance` и файл БД создаются автоматически; проверьте права на запись в папке проекта |
---

## 8. Краткая шпаргалка

```bash
# 1. Активировать окружение
venv\Scripts\activate          # Windows
source venv/bin/activate       # Linux/macOS

# 2. Запуск
python run.py

# 3. В браузере
#    — Главная:     http://127.0.0.1:5000
#    — Тестовые данные: http://127.0.0.1:5000/dev-setup
#    — Вход (dev):   http://127.0.0.1:5000/dev-login
```

После выполнения этих шагов приложение должно запускаться и быть доступным по указанным адресам.
